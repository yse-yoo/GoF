Pattern 解説: Strategy

  • 1. アルゴリズムの部品化 計算ロジックをクラスとして独立させることで、本体（Checkout）のコードを汚さずにルールを増やせます。
  • 2. 実行時の動的切り替え 「今の支払いにはこの計算機を使う」という指示を出すだけで、中身を書き換えずに挙動を変更できます。

js/strategy.js

JavaScript 
// 戦略群: 同じ「calculate」メソッドを持つ
class PercentDiscount {
    calculate(price) { return price * 0.8; }
}

class FixedDiscount {
    calculate(price) { return price - 1000; }
}

// コンテキスト: どの戦略を使うか保持する
class Checkout {
    setStrategy(strategy) { this.strategy = strategy; }
    
    calculate(price) {
        // 実際の計算はセットされた戦略に任せる
        return this.strategy.calculate(price);
    }
}
