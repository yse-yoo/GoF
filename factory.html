Pattern 解説: Factory Method

  • 1. インスタンス化の隠蔽 利用側はどのクラスを new するか知る必要がなく、Factoryに頼むだけで済みます。
  • 2. 共通インターフェース 製品が異なっても render() などの共通メソッドで一貫した操作が可能です。

js/factory.js

JavaScript 
// クリエイター (工場)
class UIFactory {
    static create(type) {
        if (type === 'info') return new InfoBadge();
        if (type === 'warning') return new WarningAlert();
    }
}

// 共通のメソッドを持つ製品たち
class InfoBadge {
    render() {
        return `<div class="bg-blue-100...">INFO</div>`;
    }
}

class WarningAlert {
    render() {
        return `<div class="bg-red-100...">WARNING</div>`;
    }
}
