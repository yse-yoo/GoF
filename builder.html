Pattern 解説: Builder

  • 1. 構築手順の分離 ディレクター（監督者）がビルダーに指示を出すことで、一貫した手順で異なる製品を作れます。
  • 2. メソッドチェーン .setColor().setIcon() のように繋げて書くことで、構築過程が直感的になります。

js/builder.js

JavaScript 
// ビルダー: 手順をメソッドとして提供
class NotificationBuilder {
    setColor(color) { this.card.color = color; return this; }
    setIcon(icon)   { this.card.icon = icon; return this; }
    build() { return this.card; }
}

// 監督者: 構築のレシピ（手順）を定義
class NotificationDirector {
    constructSuccess(builder, msg) {
        return builder
            .setColor('bg-green-100')
            .setIcon('✅')
            .build();
    }
}
