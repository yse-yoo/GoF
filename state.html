Pattern 解説: State

  • 1. if文の解消 「もし下書きなら...」という分岐をクラスに閉じ込めることで、メインロジックが肥大化するのを防ぎます。
  • 2. 状態の自己管理 各状態クラスが「次はどの状態へ遷移すべきか」を知っているため、複雑なワークフローを整理できます。

js/state.js

JavaScript 
// 状態クラス: 振る舞いと次の遷移先を定義
class DraftState {
    handle(doc) {
        // 承認待ち状態へ切り替え
        doc.setState(new ModerationState());
    }
}

// コンテキスト: 現在の状態を保持し、処理を委譲する
class DocumentContext {
    constructor() {
        this.state = new DraftState();
    }
    
    process() {
        // 今の状態クラスに「次どうするか」を丸投げ
        this.state.handle(this);
    }
    
    setState(newState) { this.state = newState; }
}
